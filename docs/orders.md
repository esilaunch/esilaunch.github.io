## Retrieve an order

**GET: `/order/:id`**

**RESPONSE**: responds with an order object.

## List all orders

**GET: `/order`**

**OPTIONS**: optional query parameters include: q, customerId, updatedAt, orderPaidDate, fulfillmentStatus, complianceStatus, paymentStatus, and channel. Query params may look like /order?channel=Web&orderPaidDate=gt:2018-01-01&paymentStatus=Paid

**RESPONSE**: responds with an array of order objects and a total count.
